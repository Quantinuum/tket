26 // NOTE: apart from gates taking a non-fixed number of qubits,
27 // the unitaries are not sparse enough
28 // to make sparse matrices worthwhile.
29 // But for calculating the unitary of a circuit,
30 // or applying the circuit unitary to another matrix,
31 // it may be advisable to use sparse matrices.
32 // (However, this would be done by basis permuting/expanding
33 // the unitary matrices returned by these,
34 // so dense->sparse routines would be needed).

37 // https://tket.quantinuum.com/api-docs/optype.html
